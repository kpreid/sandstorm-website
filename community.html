Welcome!

We're so glad you're interested in getting involved! We're a movement of app makers, users, open source enthusiasts, self-hosting fanatics, enterprise IT staffers, documentation writers, and more, working together on making server apps safe and easy for everyone to use. Here's how you can contribute.

Help others

  • Use & Share Sandstorm

    Self-host Sandstorm to install apps from the App Market. Then share by email or by sharing link to invite friends and colleagues to collaborate.

  • Give a Talk

    Get started with our speaker kit to prepare your presentation. Send us an email if you need help with finding a venue or preparing for your talk. We'll also send you some sandcat stickers to give out!

  • Discuss & Answer Questions

    Development and user help happens on the sandstorm-dev group; join us and contribute! Real-time chat happens in IRC (#sandstorm on libera.chat), with chat archives available too.

  • Build a Local Community

    You can join or create a meetup group near you. Email us to get started.

  • Write a Blog Post

    Share your Sandstorm experiences with the world by writing a blog post. Send us a link and we'll share it with the community. Email us if you need topic ideas.

Help the project

  • Package an App

    Check out our tutorial to get started packaging apps for the App Market. If you get stuck, email us for help or hop on IRC.

  • Provide Feedback

    To provide suggestions for each app: visit the Sandstorm App Market, click on the app, click on the app's Code link, then navigate to its issue tracker. You can report bugs in Sandstorm itself on our issue tracker.

  • Design App Icons

    Many apps packaged for Sandstorm need icons and design help. If you have design skills and would love to help improve apps, send us an e-mail and we'll connect you with apps in need.

  • Usability Test

    Usability tests are a great way to identify user problems and areas for improvement. Find volunteers, record them as they try Sandstorm, review the videos, then file issues. For more help read Usability 101 by Jakob Nielsen and e-mail us for tips.

  • Improve the Documentation

    The Sandstorm documentation lives in the main GitHub repository for Sandstorm. Listen to IRC and the dev group, identify a common question, then submit a pull request.

  • Contribute to the Core

    Sandstorm's code is open source and lives on GitHub. To contribute bug reports, patches, design ideas, documentation, pull requests, and more check out our guide to contributing.
